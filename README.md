# BeerTracker

A fully static beer-tracking website for a group of friends, hostable for free on GitHub Pages.

## Stack

| Layer | Technology |
|---|---|
| Frontend | React 18 + TypeScript + Vite |
| Routing | React Router v6 (HashRouter) |
| Styling | Plain CSS with CSS variables |
| Beer catalog | SQLite (sql.js WASM), loaded from `public/beers.sqlite` |
| User accounts | localStorage (SHA-256 hashed passwords via Web Crypto API) |
| Drink logs | localStorage |
| Deployment | GitHub Pages via `gh-pages` npm package |

> **Security note:** This app runs entirely client-side. Passwords are hashed (SHA-256) before storage, but anyone with access to the machine can inspect and manipulate localStorage. Do not use this for sensitive data.

## Prerequisites

- Node.js ≥ 18
- npm ≥ 9

## Local development

```bash
# 1. Install dependencies
npm install

# 2. Build the beer SQLite database (downloads ~6k beers from Open Beer Database)
npm run build-db

# 3. Start the dev server
npm run dev
```

Open http://localhost:5173 in your browser.

npm run build

## Deploying to GitHub Pages

### One-time setup

1. Create a GitHub repository (e.g. `yourusername/beertracker`).

2. Push this project to it:
   ```bash
   git remote add origin https://github.com/yourusername/beertracker.git
   git push -u origin main
   ```

3. In `vite.config.ts`, `base` is already set to `'./'` — this makes all
   asset paths relative, so the app works under any sub-path.

4. Set the deploy target in `package.json` if you want a custom repo name:
   ```json
   "deploy": "gh-pages -d dist -r https://github.com/yourusername/beertracker.git"
   ```

### Deploy

```bash
npm run deploy
```

This runs `npm run build` then pushes `dist/` to the `gh-pages` branch.
GitHub Pages will serve the site at:

```
https://yourusername.github.io/beertracker/
```

Enable GitHub Pages in your repo **Settings → Pages → Source: gh-pages branch**.

## Building the beer database manually

The `npm run build-db` script tries to download the
[Open Beer Database](https://github.com/nicholasgasior/gsr-beer) (~6 000 beers)
and writes it to `public/beers.sqlite`.

If the download fails (e.g. offline/CI), it falls back to a built-in 50-beer sample.

### Using your own JSON data

If you have a beer list as a JSON array like:

```json
[
  { "name": "Duvel", "brewery": "Duvel Moortgat", "style": "Belgian Strong Ale", "abv": 8.5, "country": "Belgium" }
]
```

You can adapt `scripts/build-db.js`:

```js
// Replace the fetchBeers() call with:
const beers = JSON.parse(readFileSync('my-beers.json', 'utf8'))
  .map((b, i) => ({ id: String(i + 1), ...b }));
```
